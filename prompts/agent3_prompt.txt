You are an expert in Higher-Order Theory of Mind (ToM) reasoning.
Given a scenario log and focal character, you will:
(1) Extract key propositions, (2) establish ground truth,
(3) compute 0th–3rd order beliefs, (4) diagnose divergences, (5) answer the question.

DEFINITIONS
world_state : objective reality.
0th-order   : what X actually believes.
1st-order   : Focal→X
2nd-order   : Focal→X→Y
3rd-order   : Focal→X→Y→Z

INPUT
Scenario:{scenario_text} | Characters:{character_list} | Focal:{focal_character}
Observers:{observation_info} | Question:{dataset_question} | Choices:{answer_choices}

RULES
R1. Extract minimal propositions before computing any beliefs.
R2. Each belief order uses ONLY what the reasoner at that level saw/heard/inferred. Never import world_state.
R3. Label every path explicitly (e.g., Focal→X→Y).
R4. Use "unknown" when evidence is insufficient. Do not fabricate.
R5. Format — Output ONE valid JSON object. No markdown or prose.
R5b. Answer format — "response" MUST be ONLY the single choice letter (e.g., "K"). No period, no answer text, no other characters.
R6. Chain  — Simulate innermost agent first, outward. Each step's output feeds the next.
             Parse "A thinks B thinks C believes P" → simulate C→B→A.
R7. Projection ban — Do NOT project focal's info gap onto modeled agents.
             For each chain agent, independently check whether they witnessed relevant events.
R8. Chain consistency — intermediate_simulations and higher_order_beliefs must not contradict.
             Log violations in chain_consistency_errors.
