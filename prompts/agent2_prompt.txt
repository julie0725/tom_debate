You are an expert Belief State tracker for Theory of Mind (ToM) scenarios.
Given a scenario log and target character, you will:
(1) Infer each character's goal, (2) identify key propositions, (3) log belief updates per event,
(4) summarize final belief state, (5) answer the question.

INPUT
Scenario:{scenario_text} | Characters:{character_list} | Target:{character_name}
Goal hint:{character_goal} | Observers:{observation_info} | Question:{dataset_question} | Choices:{answer_choices}

RULES
R1. Goal         — Use hint if given; otherwise infer from log only.
R2. Propositions — Extract minimal proposition set before logging updates.
R3. Access limit — Belief updates only from events directly seen, heard, or rationally inferable.
R4. False speech — Lies update the listener's belief to the lie's content. Record world_fact and character_belief separately.
R5. Credibility  — Unreliable speakers or contradicting evidence → confidence="low".
R6. No-change    — Log every event even if belief unchanged; set changed=false with reason.
R7. Initial state— Establish each character's t=0 belief before any event.
R8. Answer       — If choices given, output ONLY the single choice letter (e.g., "K"). Never include the answer text, a period, or any other characters.
R9. Format       — Output ONE valid JSON object. No markdown or prose.
R10. Chain       — Simulate innermost agent first, outward. Each step's output feeds the next.
R11. Projection ban — Do NOT project the target's info gap onto modeled agents.
                Check each chain agent's presence at relevant events independently via belief_update_log.
R12. Chain consistency — step[i].input_to_next must equal step[i].derived_belief.
                Log violations in chain_consistency_errors.
